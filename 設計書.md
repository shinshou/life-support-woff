# LINE WORKS × WOFF タスク管理ミニアプリ 設計書

## 1. 概要

LINE WORKS上で動作するWOFFミニアプリとして、部署横断プロジェクトのタスク管理・進捗共有・通知を行う。

| 項目 | 技術選定 |
|---|---|
| フロントエンド | WOFFミニアプリ（HTML / CSS / JavaScript） |
| バックエンド | Google Apps Script（Web App） |
| データストア | Googleスプレッドシート |
| 通知 | LINE WORKS Bot API |

---

## 5. スプレッドシート シート定義

### シート1: プロジェクト

| 列名 | 型 | 必須 | 説明 | 例 |
|---|---|---|---|---|
| project_id | String | ○ | プロジェクト一意ID（自動採番） | `proj_1712345678_abc` |
| project_name | String | ○ | プロジェクト名 | `2024年春 新人研修` |
| project_type | String | ○ | プロジェクト種別 | `研修` / `イベント` / `業務改善` |
| start_date | Date | ○ | 開始日（YYYY-MM-DD） | `2024-04-01` |

### シート2: タスク

| 列名 | 型 | 必須 | 説明 | 例 |
|---|---|---|---|---|
| task_id | String | ○ | タスク一意ID（自動採番） | `task_1712345678_xyz` |
| project_id | String | ○ | 紐付くプロジェクトID | `proj_1712345678_abc` |
| default_task_id | String | - | デフォルトタスクから生成した場合のID | `dtask_001` |
| task_name | String | ○ | タスク名 | `資料準備` |
| assignee | String | - | 担当者名 | `山田 太郎` |
| due_date | Date | - | 期日（YYYY-MM-DD） | `2024-04-10` |
| status | String | ○ | 未着手 / 進行中 / 完了 | `未着手` |
| comment | String | - | コメント・備考 | `A社向け資料を準備する` |

### シート3: デフォルトタスク

| 列名 | 型 | 必須 | 説明 | 例 |
|---|---|---|---|---|
| default_task_id | String | ○ | デフォルトタスク一意ID | `dtask_001` |
| task_name | String | ○ | タスク名テンプレート | `キックオフミーティング` |
| offset_days | Integer | ○ | 開始日からの日数オフセット（0=当日） | `0` / `7` / `14` |

### シート4: ルーム

| 列名 | 型 | 必須 | 説明 | 例 |
|---|---|---|---|---|
| room_id | String | ○ | LINE WORKSルームID（WOFFコンテキストから取得） | `room_9876543210` |
| room_name | String | ○ | ルーム名（管理用表示名） | `開発チームルーム` |

### シート5: プロジェクト_ルーム紐付け

| 列名 | 型 | 必須 | 説明 | 例 |
|---|---|---|---|---|
| project_id | String | ○ | プロジェクトID | `proj_1712345678_abc` |
| room_id | String | ○ | ルームID | `room_9876543210` |

※ project_id + room_id の組み合わせで一意

### シート6: 作成者管理

| 列名 | 型 | 必須 | 説明 | 例 |
|---|---|---|---|---|
| user_id | String | ○ | LINE WORKSユーザーID | `U_abcdef123456` |
| 作成可否 | Boolean | ○ | プロジェクト作成権限（TRUE / FALSE） | `TRUE` |

## 6. Model層 設計

### SpreadsheetUtil.gs（共通基盤）

```javascript
SpreadsheetUtil
  getSheet(sheetName)                        // シート取得
  getAllRows(sheet)                           // 全行をオブジェクト配列で返す（1行目をヘッダとして使用）
  findRowsByField(sheet, key, value)         // 特定フィールドで行検索
  appendRow(sheet, obj)                      // 行追加（ヘッダ順にマッピング）
  updateRow(sheet, rowIndex, obj)            // 行更新
  deleteRow(sheet, rowIndex)                 // 行削除
  generateId(prefix)                         // ID生成（prefix + タイムスタンプ + ランダム）
```

### ProjectModel.gs

```javascript
ProjectModel
  getAll()                                   // 全プロジェクト取得
  getById(projectId)                         // ID指定で1件取得
  getByRoomId(roomId)                        // ルームに紐づくプロジェクト一覧
  create(data)                               // プロジェクト作成（project_id自動採番）
  update(projectId, data)                    // プロジェクト更新
  delete(projectId)                          // プロジェクト削除
```

### TaskModel.gs

```javascript
TaskModel
  getByProjectId(projectId)                  // プロジェクト配下タスク一覧
  getById(taskId)                            // ID指定で1件取得
  create(data)                               // タスク作成（status デフォルト: "未着手"）
  update(taskId, data)                       // タスク更新
  delete(taskId)                             // タスク削除
  getOverdueTasks(today)                     // 期限超過タスク取得（status != "完了"）
  getTasksDueToday(today)                    // 本日期限タスク取得
  getTasksDueSoon(today, days)               // N日以内に期限が来るタスク取得
```

### DefaultTaskModel.gs

```javascript
DefaultTaskModel
  getAll()                                   // 全デフォルトタスク取得
  getById(defaultTaskId)                     // ID指定で1件取得
  create(data)                               // 作成
  update(defaultTaskId, data)                // 更新
  delete(defaultTaskId)                      // 削除
```

### RoomModel.gs

```javascript
RoomModel
  getAll()                                   // 全ルーム取得
  getById(roomId)                            // ID指定で1件取得
  create(data)                               // ルーム追加
  getByProjectId(projectId)                  // プロジェクトに紐づくルーム一覧
```

### ProjectRoomModel.gs

```javascript
ProjectRoomModel
  getByProjectId(projectId)                  // プロジェクトの紐付けルーム一覧
  getByRoomId(roomId)                        // ルームの紐付けプロジェクト一覧
  getRoomIdsByProjectId(projectId)           // room_id 配列のみ返す
  getProjectIdsByRoomId(roomId)              // project_id 配列のみ返す
  create(projectId, roomId)                  // 紐付け追加
  delete(projectId, roomId)                  // 紐付け削除
```

### UserModel.gs

```javascript
UserModel
  getAll()                                   // 全ユーザー取得
  getByUserId(userId)                        // ID指定で1件取得
  canCreate(userId)                          // 作成権限確認（true / false）
  create(userId, canCreate)                  // ユーザー追加
  update(userId, canCreate)                  // 権限更新
```

## 7. Controller層 設計

### Code.gs（エントリポイント）

```javascript
function doGet(e) {
  // e.parameter.action が未指定 → index.html を返す（HtmlService）
  // e.parameter.action が存在  → Router.routeGet(e) へ委譲
}

function doPost(e) {
  // Router.routePost(e) へ委譲
  // 全レスポンスは ContentService.createTextOutput(JSON) で返す
}
```

### GETエンドポイント一覧

| action | 説明 | 必須パラメータ |
|---|---|---|
| `getProjects` | ルームに紐づくプロジェクト一覧取得 | `roomId` |
| `getTasks` | プロジェクト配下タスク一覧取得 | `projectId`, `roomId` |
| `getDefaultTasks` | デフォルトタスク一覧取得 | なし |
| `getRooms` | ルーム一覧取得 | なし |
| `getProjectRooms` | プロジェクトに紐づくルーム一覧取得 | `projectId` |

### POSTエンドポイント一覧

| action | 説明 | 権限 |
|---|---|---|
| `createProject` | プロジェクト作成 | 作成者のみ |
| `updateProject` | プロジェクト更新 | 作成者のみ |
| `deleteProject` | プロジェクト削除 | 作成者のみ |
| `createTask` | タスク個別作成 | ルーム参加者 |
| `updateTask` | タスク更新（完了時はBot即時通知） | ルーム参加者 |
| `deleteTask` | タスク削除 | ルーム参加者 |
| `createDefaultTasks` | デフォルトタスク一括展開 | 作成者のみ |
| `linkRoom` | プロジェクトへルーム紐付け | 作成者のみ |
| `unlinkRoom` | プロジェクトからルーム解除 | 作成者のみ |

### 共通レスポンス形式

```json
// 成功
{ "success": true, "data": { ... }, "error": null }

// 失敗
{ "success": false, "data": null, "error": "エラーメッセージ" }
```

### アクセス制御フロー（全エンドポイント共通）

```
リクエスト受信
  ↓
AuthService.verifyAccess(userId, roomId, projectId)
  ├─ RoomModel でroomId存在確認
  ├─ ProjectRoomModel でプロジェクトとルームの紐付け確認
  └─ NG → 403エラーレスポンス返却
  ↓
createProject系のみ: AuthService.canCreateProject(userId)確認
  └─ NG → 403エラーレスポンス返却
  ↓
対応するServiceメソッドを呼び出す
  ↓
ResponseUtil.success(data) または ResponseUtil.error(message) を返す
```

## 9. WOFF認証フロー

### 認証シーケンス

```
[LINE WORKSアプリ] → WOFFミニアプリを開く
        ↓
[index.html ロード]
        ↓
woff.init({ woffId: "WOFF_ID" })
        ↓ 成功
woff.getContext() → { type, roomId, ... }
        ↓
woff.getProfile() → { userId, displayName, ... }
        ↓
取得した userId と roomId をメモリに保持
（LocalStorageは使用しない: セキュリティ考慮）
        ↓
APIリクエスト時に毎回 userId・roomId をリクエストボディに付与
        ↓
[GAS: AuthService.verifyAccess(userId, roomId, projectId)]
  ├─ RoomModel でroomId存在確認
  ├─ ProjectRoomModel でプロジェクトとルームの紐付け確認
  └─ OK → 処理続行 / NG → 403 返却
```

### AuthService.gs 主要関数

```javascript
AuthService
  verifyAccess(userId, roomId, projectId)
    // roomIdの存在確認 → projectIdとroomIdの紐付け確認
    // 失敗時: エラーをthrow

  canCreateProject(userId)
    // UserModel.canCreate(userId) を呼び出し boolean を返す

  extractContext(e)
    // doPost の e.postData.contents をパースし
    // { userId, roomId, action, ...params } を返す
```

### セキュリティ上の注意事項

- WOFF SDK から取得した userId はサーバーサイドで都度検証する
- スクリプトプロパティに認証情報を格納し、ソースコードに直書きしない
- すべてのエンドポイントにアクセス制御ロジックを適用する
- 認証強化が必要な場合: `woff.getAccessToken()` をGAS側でLINE WORKS APIにより検証する

## 10. 通知処理 設計

### 通知トリガー種別

| 種別 | タイミング | 方式 |
|---|---|---|
| タスク完了通知 | ステータスが「完了」に変更された瞬間 | API即時呼び出し |
| 期限当日通知 | due_date 当日 AM 8:00 | 時間駆動トリガー |
| 期限前通知（3日前） | due_date の3日前 AM 8:00 | 時間駆動トリガー |
| 遅延通知 | due_date を超過 かつ status != "完了" の毎朝 | 時間駆動トリガー |

### DailyTrigger.gs（スケジューラ）

```javascript
function runDailyNotification() {
  // 毎日 AM 8:00 に実行
  const today = DateUtil.getToday();

  // 遅延タスク通知
  checkOverdueTasks(today);
  // → TaskModel.getOverdueTasks(today) → 各タスクに NotificationService.sendOverdue(task)

  // 本日期限タスク通知
  checkDueTodayTasks(today);
  // → TaskModel.getTasksDueToday(today) → 各タスクに NotificationService.sendDueToday(task)

  // 期限3日前通知
  checkDueSoonTasks(today, 3);
  // → TaskModel.getTasksDueSoon(today, 3) → 各タスクに NotificationService.sendDueSoon(task, daysLeft)
}
```

### NotificationService.gs

```javascript
NotificationService
  sendTaskComplete(task)    // "タスク完了: {task_name}（{project_name}）"
  sendDueToday(task)        // "本日期限: {task_name}（担当: {assignee}）"
  sendDueSoon(task, days)   // "期限{days}日前: {task_name}（担当: {assignee}）"
  sendOverdue(task)         // "期限超過: {task_name}（期限: {due_date}）"

  _getRoomsForTask(task)    // ProjectRoomModel.getRoomIdsByProjectId() でルームID取得
  _postToRoom(roomId, msg)  // LINE WORKS Bot API へ POST
```

### 時間駆動トリガー設定

```
GASプロジェクト → 「トリガー」→ トリガーを追加

  実行関数    : runDailyNotification
  イベントソース: 時間主導型
  時間ベース  : 1日タイマー
  時刻       : 午前8時〜午前9時
```

### LINE WORKS Bot API 認証設定

スクリプトプロパティに以下を登録する:

| キー | 内容 |
|---|---|

function createDefaultTasks(projectId, startDate, selectedDefaultTaskIds) {
  selectedDefaultTaskIds.forEach(id => {
    const defaultTask = DefaultTaskModel.getById(id);
    const dueDate = DateUtil.addDays(startDate, defaultTask.offset_days);
    TaskModel.create({
      project_id: projectId,
      default_task_id: defaultTask.default_task_id,
      task_name: defaultTask.task_name,
      due_date: dueDate,
      status: "未着手"
    });
  });
}
